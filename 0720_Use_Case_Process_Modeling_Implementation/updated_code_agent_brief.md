# 更新後的程式碼代理實作任務書（Updated Code Agent Brief）

一、專案與目前狀態

- 專案名稱：校園宿舍報修、派工與維修追蹤系統
- 目前系統狀態：第一個可操作切片已具備學生報修、管理員派工、維修人員完成回報的三角色前端流程
- 本次基準：0714 需求文件包初稿、0713 候選需求、AC-01-01 到 AC-03-02

二、本次使用案例與目標

- UC-01：提交報修
- UC-03：派工給維修人員
- UC-05：完成維修回報

三、本次要完成的流程

- 主要流程：學生送出報修 → 管理員派工 → 維修人員完成回報
- 替代流程：表單缺漏時補正後再送出
- 例外流程：必填欄位缺漏或案件狀態異常時停止操作並顯示訊息

四、前置與後置條件

- 前置：案件列表可被管理員或維修人員看到，且狀態處於正確階段
- 後置：狀態依序由已提交、已派工、已完成變化，並保留可理解結果

五、輸入、輸出與狀態變化

- 輸入：宿舍、房號、設備類型、問題描述、維修人員名稱
- 輸出：案件編號、狀態標籤、完成結果
- 狀態變化：待提交 → 已提交 → 已派工 → 已完成

六、保留項目

- 角色切換介面
- 既有假資料被 localStorage 保存
- 繁體中文操作文字

七、修正項目

- 補強缺欄位驗證
- 補強狀態文字與清單顯示
- 保留學生送出、管理員派工、維修人員完成的核心流程

八、新增項目

- 清楚的成功提示與錯誤提示
- 狀態標籤統一顯示

九、刪除項目

- 正式登入
- 正式資料庫
- 手機 App
- 金流與採購

十、本次不做

- 正式單一登入
- 外部通知
- 權限管理系統
- 重複報修辨識

十一、技術與資料限制

- HTML、CSS、JavaScript
- 僅使用前端假資料與 localStorage
- 不做正式部署

十二、對應需求與驗收條件

- FR-01、FR-02、FR-03、FR-04、FR-05、NFR-01
- AC-01-01、AC-01-02、AC-01-03、AC-02-01、AC-02-02、AC-03-01、AC-03-02、AC-NFR-01-01

十三、不可自行判斷事項

- 不得自行新增正式登入角色、資料庫或通知功能
- 不得擴大為完整宿舍管理系統
- 重複報修辨識、時間欄位保存待後續需求確認

十四、完成後回報

1. 修改檔案與用途
2. 操作方式
3. 對應使用案例、流程與驗收條件
4. 已完成與未完成待辦
5. 已知限制
6. 需要人工驗收之處
