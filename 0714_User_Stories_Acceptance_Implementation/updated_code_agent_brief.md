# 更新後的程式碼代理實作任務書（Updated Code Agent Implementation Brief）

一、專案名稱
校園宿舍報修、派工與維修追蹤系統

二、目前系統狀態
目前原型已具備：學生報修提交、角色切換、管理員待處理清單、派工操作、維修人員完成回報、假資料 localStorage 儲存。

三、本次目標
依需求與驗收條件，完成一個可操作的第一個切片修正版，確保從學生送出報修到管理員派工再到維修人員完成回報的流程可重現。

四、保留項目
1. 既有角色切換介面與三個主要畫面
2. 既有假資料格式與 localStorage 儲存方式
3. 繁體中文操作文字

五、修正項目
1. 補齊必填欄位驗證與錯誤訊息
2. 明確顯示案件狀態與派工結果
3. 讓維修人員完成動作時狀態可從「已派工」更新為「已完成」

六、新增項目
1. 新增清楚的報修成功提示
2. 新增管理員派工與維修人員任務清單的狀態顯示

七、刪除項目
1. 不納入正式登入、正式資料庫、採購、金流、手機 App 等範圍外功能

八、本次不做
1. 正式單一登入
2. 正式資料庫整合
3. 電子郵件或簡訊通知
4. 採購與金流
5. 重複報修辨識（待後續需求確認）

九、技術與資料限制
1. 使用 HTML、CSS、JavaScript
2. 僅使用前端假資料與 localStorage
3. 不做正式部署與後端資料儲存
4. 介面使用繁體中文

十、對應需求與驗收條件
- FR-01：AC-01-01、AC-01-02
- FR-02：AC-01-03
- FR-03：AC-02-01
- FR-04：AC-02-02
- FR-05：AC-03-01
- FR-06：AC-03-02
- NFR-01：AC-NFR-01-01

十一、實作順序
1. 先修正報修表單驗證與送出結果
2. 再完成管理員清單派工與狀態更新
3. 最後完成維修人員任務確認與完成回報

十二、不可自行決定的事項
1. 不得新增正式登入角色或真實帳號資料
2. 不得新增金流、採購或電子通知功能
3. 不得擴大為完整宿舍管理系統

十三、完成後回報格式
1. 修改檔案與用途
2. 操作方式
3. 需求與驗收條件對應
4. 已通過與未通過項目
5. 已知限制
6. 未完成項目
